# Proxy Networks

Chrome MV3-расширение для управления прокси-профилями: включение/выключение, выбор профиля, поддержка fixed servers, PAC-скриптов и базовой авторизации.

## Возможности

- Несколько прокси-профилей
- Переключение режима прокси из popup
- Поддержка:
  - `direct`
  - `auto_detect`
  - `system`
  - `fixed_servers`
  - `pac_script`
- Авторизация прокси через `webRequestAuthProvider`
- Раздельная область для инкогнито (`incognito: split`)
- Хранение профилей в `chrome.storage.local`
- Обфускация пароля AES-GCM на уровне локального хранилища
- Базовая проверка текущего IP через `api.ipify.org`
- Импорт и экспорт профилей в JSON
- Расширенная диагностика ошибок и подсказки по настройке

## Установка

1. Откройте `chrome://extensions`
2. Включите режим разработчика
3. Нажмите **Загрузить распакованное расширение**
4. Выберите папку репозитория

После загрузки откройте popup или страницу настроек расширения.

## Зачем нужны разрешения

- `proxy` — применение конфигурации прокси через `chrome.proxy.settings`
- `webRequest` — обработка запросов
- `webRequestAuthProvider` — ответ на proxy-auth challenge в MV3
- `storage` — сохранение профилей и настроек
- `host_permissions: <all_urls>` — возможность работать со всеми целевыми URL и выполнять health-check

## Ограничения MV3

- Расширение настраивает прокси на уровне профиля браузера, а не отдельной вкладки.
- Точечная маршрутизация по доменам возможна через PAC-скрипт.
- `webRequest` в MV3 не используется для перехвата/модификации потоков; здесь он нужен только для proxy auth.
- Диагностику прокси удобно смотреть в `chrome://net-internals/#proxy`.

## Структура файлов

- `manifest.json` — манифест расширения
- `background.js` — service worker
- `storage.js` — CRUD профилей и шифрование пароля
- `popup.html`, `popup.js`, `popup.css` — быстрый переключатель
- `options.html`, `options.js`, `options.css` — редактор профилей
- `icons/` — иконки расширения

## Примечания

Пароль сохраняется в зашифрованном виде AES-GCM, но ключ тоже хранится рядом в локальном хранилище. Это соответствует лёгкому уровню защиты и удобству восстановления профилей на одном устройстве.

Экспорт профилей можно выполнить как без паролей, так и вместе с паролями. Экспорт с паролями используйте только для доверенного локального переноса.
